# WidgetKit Build Guide (macOS)

## Overview
This repo ships WidgetKit source files under `mac-widget/` and an Xcode project generated by XcodeGen.

## Requirements
- Xcode 15+
- macOS 13+
- XcodeGen (already installed on this machine)

## Steps
1. Use the generated project at `mac-widget/OpenUsageWidget.xcodeproj`.
2. Ensure App Group entitlements are enabled for both targets (already set in `.entitlements` files).
3. If you need a specific signing team, set `DEVELOPMENT_TEAM` in `mac-widget/project.yml` and re-run `xcodegen`.
4. Confirm the widget reads the shared file:
   - `~/Library/Group Containers/group.ai.openusage.local/usage.json`

## Local Build / Run
1. Build and run the app target once (it installs the widget extension).
2. In macOS: add the “OpenUsage” widget from the widget gallery.
3. Verify the widget refreshes every ~15 minutes.

### CLI Build (no Xcode UI)
```
cd /Users/R/Work/openusage/mac-widget
xcodebuild -project OpenUsageWidget.xcodeproj -scheme OpenUsageWidgetApp -configuration Debug build
```
If code signing fails, set `DEVELOPMENT_TEAM` in `project.yml`, run `xcodegen`, and retry.

## Signing / Distribution
- Use a single Apple Developer Team for both targets.
- Ensure the App Group entitlement matches `group.ai.openusage.local` across targets.
- Archive the app in Xcode, then distribute as needed.

## Troubleshooting
- If the widget shows “No data,” confirm `usage.json` exists and the App Group ID is correct.
- If the widget fails to load, ensure the widget extension target is included in the scheme build phases.
